# HTTP 协议
## 基本概念:HTTP 请求方法概述
HTTP 请求方法是客户端与服务器之间通信的方式，它们定义了对给定资源执行的期望动作。这些方法有时被称为 HTTP 动词，每个方法都有其特定的用途和语义。
### 常见的 HTTP 请求方法
  GET: 请求获取指定资源的表示形式，通常用于获取数据，如网页或图片。
  
  HEAD: 类似于 GET 方法，但服务器只返回响应头，不包含响应体，用于获取资源的元数据。
  
  POST: 向服务器提交数据以创建或更新资源，常用于表单提交或文件上传。
  
  PUT: 用于替换目标资源的所有当前表示，如果资源不存在，则创建新资源。
  
  DELETE: 删除指定的资源。
  
  CONNECT: 建立到目标资源标识的服务器的隧道，常用于 HTTPS 连接。
  
  OPTIONS: 描述目标资源的通信选项，常用于跨域资源共享（CORS）的预检请求。
  
  TRACE: 执行消息环回测试，用于诊断。
  
  PATCH: 对资源进行部分修改，只更改资源的一部分而不是整个资源
### HTTP方法名是可以自定义的
  在真实 Web 开发中，HTTP 方法名是可以自定义的，但大多数服务器会拒绝非标准方法。  
  -X 是 curl 的“自定义请求方法”参数，curl -X 可以强行发送任意方法  
  如果你还想深入看抓包效果，可以用 curl -v  
  即curl -v -X 自定义的请求方法 要访问的网址
